Distributed File Server with gRPC

C++ Remote Procedure Calls Multi-threaded Programming Object Oriented Programming
  • Implemented a distributed file system based on the Andrew File System using C++ and remote procedure calls (gRPC).
  • Using a weakly consistent file caching strategy, any client can store, fetch, delete, and list all files with guaranteed eventual synchronization.
  • Multi-threaded and multi-process design with per-file locking mechanism combined with a streaming RPC with deadline handling ensures data transfer speed and integrity.
